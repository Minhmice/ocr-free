# Codebase Structure

**Analysis Date:** 2026-05-06

## Directory Layout

```
mineru/
├── mineru/            # Python package: CLI, backend engines, models, data adapters, utilities
├── tests/             # End-to-end style unittest harness and coverage helpers
├── docs/              # MkDocs content and static assets
├── docker/            # Deployment Dockerfiles and compose
├── demo/              # Sample documents and demo script
├── .github/workflows/ # CI workflows
└── pyproject.toml     # Build metadata, dependencies, entrypoints, test config
```

## Directory Purposes

**`mineru/cli/`:**
- Purpose: user-facing command and service entrypoints.
- Contains: CLI command handlers, API server, router, API client, output path helpers.
- Key files: `mineru/cli/client.py`, `mineru/cli/fast_api.py`, `mineru/cli/router.py`.

**`mineru/backend/`:**
- Purpose: backend-specific orchestration and output normalization.
- Contains: `pipeline`, `vlm`, `hybrid`, `office` analyze and conversion modules.
- Key files: `mineru/backend/pipeline/pipeline_analyze.py`, `mineru/backend/vlm/vlm_analyze.py`.

**`mineru/model/`:**
- Purpose: model wrappers and format converters.
- Contains: OCR/layout/table/formula/VLM and office converters.
- Key files: `mineru/model/ocr/pytorch_paddle.py`, `mineru/model/docx/main.py`, `mineru/model/vlm/vllm_server.py`.

**`mineru/data/`:**
- Purpose: storage and transport adapters.
- Contains: generic IO readers/writers for file/http/s3.
- Key files: `mineru/data/io/base.py`, `mineru/data/io/http.py`, `mineru/data/io/s3.py`.

**`mineru/utils/`:**
- Purpose: shared cross-cutting helper functions.
- Contains: config/environment parsing, PDF/image helpers, utility logic.
- Key files: `mineru/utils/config_reader.py`, `mineru/utils/cli_parser.py`.

## Key File Locations

**Entry Points:**
- `pyproject.toml`: defines scripts `mineru`, `mineru-api`, `mineru-router`, server/model utilities.
- `mineru/cli/client.py`: primary CLI execution path.

**Configuration:**
- `pyproject.toml`: dependencies and test/coverage options.
- `mineru.template.json`: example runtime config template.
- `mineru/utils/config_reader.py`: runtime env + local config file access.

**Core Logic:**
- `mineru/backend/`: engine orchestration and post-processing.
- `mineru/model/`: inference and format conversion.

**Testing:**
- `tests/unittest/test_e2e.py`: end-to-end parse scenario.
- `tests/get_coverage.py`, `tests/clean_coverage.py`: coverage workflow helpers.

## Naming Conventions

**Files:**
- Snake case modules: `pipeline_analyze.py`, `model_output_to_middle_json.py`.
- Package partitioning by domain/backend under `mineru/backend/` and `mineru/model/`.

**Directories:**
- Domain-first layout: `cli`, `backend`, `model`, `data`, `utils`.

## Where to Add New Code

**New Feature:**
- Primary code: place engine logic in `mineru/backend/<engine>/` and model logic in `mineru/model/<domain>/`.
- Tests: add scenarios under `tests/unittest/`.

**New Component/Module:**
- Implementation: add user-facing commands/services under `mineru/cli/` and keep shared primitives in `mineru/utils/`.

**Utilities:**
- Shared helpers: use `mineru/utils/` for generic helpers; use `mineru/backend/utils/` for backend-only helpers.

## Special Directories

**`mineru/resources/`:**
- Purpose: packaged static resources (for example language detection model files and HTML templates).
- Generated: No
- Committed: Yes

**`mineru.egg-info/`:**
- Purpose: package metadata generated by builds/installs.
- Generated: Yes
- Committed: Yes (currently present in repository state)
